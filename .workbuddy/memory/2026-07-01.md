## 任务一重写：基础几何体版（16:40）

### 问题诊断
用户反馈首版 3D 模型有"多余面、闭合错误"。根因三处叠加：
1. ExtrudeGeometry + Shape.holes（眼环/外环/openwork）孔路径方向错→法线翻转
2. ProfileSweep 沿 border 扫掠几何与主结构 Extrude 重叠穿插→多余面
3. horseshoe CurvePath 拼接处不闭合

### 文献代码调研结论
两篇文献均无可直接移植到 WebGL/JS 的现成代码：
- 文献一：Scheme+POV-Ray，论文仅贴 ~15行 rose 函数 cond 分支
- 文献二：GML 自定义栈语言，论文贴 ~30行片段，完整代码在失效的 2004 链接

### 重写方案：基础几何体组合（文献二精神）
放弃 ExtrudeGeometry+Shape+ProfileSweep，改用 Three.js 内置几何体：
- 眼环/外环 = TorusGeometry（数学完美闭合）
- N辐柱 = CylinderGeometry（径向旋转放置）
- 花叶 = TorusGeometry 弧段（regular整圆/semicircular半圆/segmental弓形/horseshoe三段组合）
- 拱肩 = 小 TorusGeometry 多叶组合（trefoil 3叶/quatrefoil 4叶/openwork+中心环）

每个部件独立网格，无自交、无闭合错误、法线自动正确。
ProfileSweep.js / FoilShapes.js 保留为休眠文件（index.html 仍加载但不引用）。
保留文献一 7 步决策树参数 + 黄金分割比例。

### 关键修复
- _add 方法支持 BufferGeometry 和 Object3D(Group) 两种输入
- switch case 内 const 加块作用域花括号防 ES6 语法错误
