## 玫瑰窗数字展览系统 — M8 进阶：远距离粒子消散+实体模型显现

### 概念："距离=物质，靠近=光"
远离玫瑰窗(>24m)时光粒子交叉溶解消散，物理GLB模型同步显现。反转"靠近=更多细节"预期。

### 实现
- PointShader片元加 `uGlobalOpacity` uniform，alpha乘以它
- LightPointSystem.update() 按距离smoothstep算消散因子(24m→1.0, 30m→0.0)，设uGlobalOpacity + points.visible
- LightPointSystem.getDissipationFactor() 暴露给Application
- Application._initLightPoints 初始化模型材质transparent+opacity=0，存_modelMeshes引用
- Application._update 按消散因子控制模型opacity(1-factor)，factor=1时模型隐藏
- Config: boundary 25→30m + dissipation{startDistance:24, endDistance:30}

### 选择性Bloom兼容
- 模型是Mesh，Bloom阶段换黑不发光——正确(实体窗不该Bloom)
- 粒子uGlobalOpacity→0时Bloom贡献自然减少
- 过渡期：实体窗在base层+发光粒子additive叠加="光覆盖实体褪去"

## 玫瑰窗数字展览系统 — M4 调参迭代 2（选择性Bloom+独立粒子数）

### 选择性 Bloom（解决地板 Bloom 问题）
- 问题：浅灰地板亮度~0.58 超过 Bloom 阈值 0.25，地板严重 Bloom
- 解法：材质变暗选择性 Bloom——渲染 Bloom 前把所有 Mesh 材质换黑（THREE.Points 不是 Mesh 不受影响），Bloom 完恢复，再正常渲染全场景叠上
- 渲染流程：darkenNonBloomed→bloomComposer.render()(RenderPass+UnrealBloomPass+OutputPass)→restoreMaterial→renderer.render(autoClear=false+clearDepth)全场景叠上
- RenderLoop 改用 renderFn 回调替代 composer.render()

### 独立粒子数（Glass 可单独加）
- meshSampling 从权重(weight)改为每 mesh 独立 count
- Glass 120000 / Frame 30000（总 15 万），可单独调玻璃不影响框架

### 关键认知（可复用）
- **选择性 Bloom 标准技法**：Mesh 材质换黑（Points/Line 不受 isMesh 检查影响）→ Bloom → 恢复 → 全场景 renderer.render 叠上(autoClear=false+clearDepth)
- **Mesh vs Points 的 isMesh 检查**天然区分了"不该 Bloom 的实体"和"该 Bloom 的粒子"

## 玫瑰窗数字展览系统 — M4 优化（光感强化）

### 用户反馈驱动的迭代
用户反馈：光点轮廓✓、颜色✓、位移✓，但**发光感弱不像光粒、粒子数量不足**。
重要澄清：用户原想增加模型顶点数到70万来改善，我解释了**顶点数≠粒子数**（MeshSurfaceSampler按面积采样，粒子数由config.totalCount决定），说服用户不动模型改代码。

### 优化内容
1. 粒子 5万→10万（totalCount）
2. 片元着色器重写：高斯软边 exp(-dist²×14) + 中心增亮溢出(1+core×2.0，可达3×基色) + 新增uCoreBoost uniform
3. 点变大 size 0.03→0.055
4. **Bloom后处理**：新建PostProcessing.js（EffectComposer+RenderPass+UnrealBloomPass+OutputPass）；strength0.65/radius0.5/threshold0.12
5. 渲染管线改造：RenderLoop支持composer分支、ResizeHandler同步composer.setSize、Application集成
6. tone mapping去重：EffectComposer渲染到renderTarget时renderer强制NoToneMapping，OutputPass末端应用一次（已验证不重复）
7. 地板白→浅灰(0xccccc)、灯光小幅调低(ambient0.22/dir0.6/hemi0.15/hdr0.4)衬托光粒

### 关键认知（可复用）
- **模型顶点数≠采样粒子数**：MeshSurfaceSampler按三角面面积均匀采样，粒子数完全由调用次数决定
- **AdditiveBlending+中心溢出+Bloom**是"像光"的三件套：点颜色>1.0→Bloom捕获亮核→辉光
- **three.js后处理tone mapping去重**：renderTarget渲染时NoToneMapping，OutputPass末端读renderer设置应用一次，Renderer.js无需改
